8 Existe-t-il plusieurs types de tableaux en PHP, si oui lesquels ?

  • Tableaux associatifs : Dans ce type de tableau, les éléments sont associés à des clés plutôt qu'à des indices numériques. Chaque élément est constitué d'une paire clé-valeur.

    $personne = array(
    "nom" => "Doe",
    "prénom" => "John",
    "âge" => 25);

    echo $personne["nom"]; // Affiche "Doe"

  • Tableaux multidimensionnels : Ce sont des tableaux qui contiennent d'autres tableaux à l'intérieur. Ils permettent de représenter des structures de données complexes.

    $matrice = array( array(1, 2, 3), array(4, 5, 6), array(7, 8, 9) );

    echo $matrice[1][2]; // Affiche 6

Architecture

1 Qu’est-ce que l’architecture client / serveur ?

Elle répartit les tâches entre les fournisseurs d'un service, appelés Serveurs, et les consommateurs du service, appelés Clients

2 Donner la définition d’un design pattern. Citer au moins 3 exemples de design pattern

Un design pattern est une solution à un problème de conception récurrent.

  • Singleton : ce pattern est très couramment utilisé pour garantir qu’une classe n’a qu’une seule instance
  • Factory Method : ce pattern est couramment utilisé pour créer des objets dans une superclasse, mais permettre aux sous-classes de modifier le type d’objets qui seront créés
  • Observer : ce pattern est couramment utilisé pour informer un objet lorsqu’un événement se produit.

3 Qu’est-ce que l’architecture MVC ?

L'architecture MVC (model vue controlleur) est composée de trois responsabilités différentes:

  • Les modéles (Model) contient les données à afficher.
  • Les vues (view) contient la présentation de l'interface utilisateur.
  • Les controlleur (controller) contient la logique de controle.

4 Quel est le rôle de chaque couche du design pattern MVC : Model, View, Controller ?

  • le modèle (Model),Le modèle contient les données de l’application.
  • la vue (View) La vue est responsable de l’affichage des données et de l’interface utilisateur.
  • .Le contrôleur est responsable de la gestion des événements utilisateur et de la communication entre la vue et le modèle

5 Quels sont les avantages de l’architecture MVC ?

  • La séparation des préoccupations : chaque composant est responsable d’une tâche spécifique, ce qui facilite la maintenance et l’evolution du code.
  • La réutilisation du code : les composants peuvent être réutilisés dans d’autres projets ou parties du même projet.
  • La facilité de test : chaque composant peut être testé indépendamment des autres.

6 Existe-t-il des variantes à l’architecture MVC ?

  • MVVM (Modèle-Vue-VueModèle) : cette variante est utilisée pour les applications qui nécessitent une liaison de données bidirectionnelle entre la vue et le modèle.
  • MVP (Modèle-Vue-Présentateur) : cette variante est utilisée pour les applications qui nécessitent une séparation plus stricte entre la vue et le modèle.
  • MC (Modèle-Contrôleur) : cette variante est utilisée pour les applications qui ne nécessitent pas de vue.

RGPD

1 Qu’est-ce que le RGPD ?

Il s'agit d'un texte réglementaire européen qui encadre le traitement des données de manière égalitaire sur tout le territoire de l’Union européenne (UE).

2 Quel est son objectif principal ?

Les 3 objectifs principaux

  1. Renforcer le droit des citoyens quant au traitement de leurs données personnelles.
  2. Créer un cadre légal simplifié et unifié à l’échelle européenne.
  3. Responsabiliser davantage les entreprises sur la question du traitement des données

3 Quelle est la date d’entrée en vigueur du RGPD ?

Il est entré en application le 25 mai 2018.

4 Quelles sont les sanctions possibles en cas de non-respect du RGPD ?

Le RGPD prévoit des sanctions allant jusqu’à 20 millions d’euros ou 4% du chiffre d’affaires mondial d’une entreprise. Ces sanctions peuvent être rendues publiques.

5 En France, quel est l’autorité administrative qui s’occupe de faire appliquer le RGPD ?

En France, l’autorité administrative qui s’occupe de faire appliquer le RGPD est la Commission nationale de l’informatique et des libertés (CNIL).

6 Quel est le consentement valide selon le RPGD ?

Le consentement RGPD est une base légale pour le traitement des données à caractère personnel. Il doit être une manifestation de volonté, libre, spécifique, éclairée et univoque de la personne concernée. Il peut être donné par une déclaration ou par un acte positif clair

7 Qu’est-ce qu’une politique de confidentialité ?

Un document expliquant en détail les modalités de traitement des données personnelles dans le cadre d’une relation commerciale.

8 Quelle est la durée de conservation maximale des données personnelles selon le RGPD ?

Les données personnelles ne peuvent être conservées indéfiniment. Si elles sont inactives depuis 36 mois doivent être supprimées de la base de données. Possibilite de garde plus longtemps dans cadre d’un e-commerce car les facture doivent être conserve 10ans

9 Quels sont les droits des utilisateurs selon le RGPD ?

  • Droit d’information
  • Droit de rectification, d’effacement
  • Les droits à la limitation du traitement
